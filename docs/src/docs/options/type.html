options.type

Specifies the type of binding that this component supports. Skate offers you three different types:

  • skate.type.ELEMENT (default) - Bind to an element whose tag name matches the component name.
  • skate.type.ATTRIBUTE - Bind to an element who has an attribute matching the component name.
  • skate.type.CLASSNAME - Bind to an element who has a class name matching the component name.

You specify these as the type option:

ELEMENT

The ELEMENT type is the default and preferred type to use because it is the most explicit form of binding behaviour to an element. You should use this form unless you absolutely need to use the other types.

Use this form when:

  • You're not sure what form you should use.
  • Your component is templated.
  • Your element has semantic meaning to the beahviour in which is bound to it.

You should hyphenate your element names whenever possible because non-hyphenated names prevent Skate from using the native custom element implementation and it makes it clear to others that it is a custom element. Try to only use non-hyphenated names when you're polyfilling elements for browsers that don't support them such as datalist in IE9.

ATTRIBUTE

Use the ATTRIBUTE component type when:

  • You have functionality that you want to declaratively mix-in to your element.
  • Your component won't conflict with other components on the same element.
  • Your component does not template the element.
  • You are converting legacy components to Skate components and for whatever reason you can't change the markup.

CLASSNAME

The CLASSNAME component type is the least preferred method of binding behaviour to an element as it's not clear that an element may have custom behaviour applied to it.

You should use this form when:

  • You are converting legacy components to Skate components and for whatever reason you can't change the markup.
  • You want to hook into a particular class for non-behavioural purposes such as deprecating classes.
